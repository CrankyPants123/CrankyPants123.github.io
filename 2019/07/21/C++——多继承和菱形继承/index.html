菱形继承

概念：

  • 两个派生类继承自同一基类
  • 某个类又继承了这俩派生类

这种继承就被称作菱形继承，或者钻石继承。

解决办法：加作用域呗。

不过还有个更加高大上的解决办法——虚继承！

在继承方式前边加一个关键字virtual这个继承就变成了虚继承。

Base被称为：虚基类

这里有一个概念，叫做vbptr虚基类指针（virtual base point）。它会指向vbtable虚基类表（virtual base table）

虚基类的原理就是，Inherit1和Inherit2虽然都是Base的派生类，但是都没有真正的去继承基类的数据。而是继承了两个指针，两个指针会通过偏移量来找到唯一的数据。
